Partie Intelligence Artificielle — Traitement du Langage Naturel (NLP)

Projet Backend Spring Boot — Système de gestion des règles douanières | Sabrina Riahi

1. Objectif de la fonctionnalité

L'objectif est de permettre à l'utilisateur de saisir une règle métier douanière en langage naturel français, et que le système la transforme automatiquement en :

  • Une règle structurée (code, nom, action, conditions)
  • Un code DRL (Drools Rules Language) prêt à être exécuté par le moteur de règles

2. Architecture de la solution

Utilisateur (Angular) │ │ POST /api/nlp/convertir │ { "phrase": "Si la valeur dépasse 1 000 000 FCFA..." } ▼ NlpController │ ▼ NlpService ├── Étape 1 : Construire le prompt métier ├── Étape 2 : Appeler l'IA (Groq / LLaMA 3.3 70B) ├── Étape 3 : Parser le JSON retourné par l'IA └── Étape 4 : Générer le code DRL │ ▼ Réponse JSON → Angular (règle + DRL + score de confiance)

3. Les 4 étapes du traitement

1
Construction du prompt intelligent
On n'envoie pas seulement la phrase à l'IA. On construit un prompt enrichi qui lui fournit tout le contexte métier douanier : les catégories disponibles (TAXE, QUOTA, CONTROLE, DOUANE, CERTIFICATION, VERIFICATION), les champs de données associés à chaque catégorie, ainsi que les actions possibles. L'IA sait ainsi dans quel domaine elle travaille et peut produire une réponse cohérente.
2
Appel à l'IA via l'API Groq
Le prompt est envoyé à l'API Groq qui exécute le modèle LLaMA 3.3 70B (un grand modèle de langage open-source développé par Meta, avec 70 milliards de paramètres). L'appel est fait en HTTP depuis Spring Boot via RestTemplate. L'IA analyse la phrase et retourne un JSON structuré.
3
Parsing de la réponse JSON
La réponse brute de l'IA est parsée avec la bibliothèque Jackson. Un mécanisme de nettoyage extrait le JSON même si l'IA ajoute du texte autour (blocs ```json``` ou phrases d'introduction). Les données sont ensuite mappées vers les objets Java : NlpRegleDto, NlpConditionDto.
4
Génération du code DRL
À partir de la règle parsée, le service génère automatiquement le code Drools (.drl) prêt à être injecté dans le moteur de règles métier. Ce code contient les conditions (clause when) et l'action à déclencher (clause then).

4. Exemple concret

Entrée — Phrase saisie par l'utilisateur :

"Si la valeur de la marchandise dépasse 1 000 000 FCFA, calculer les droits de douane"

Réponse retournée par l'IA (JSON) :

{ "regle": { "code": "CALCUL_DROITS_VALEUR_ELEVEE", "nom": "Calcul droits marchandise supérieure à 1M FCFA", "action": "CALCULER_DROITS", "categorieType": "TAXE", "conditions": [ { "champ": "valeur_marchandise", "operateur": ">", "valeur": "1000000" } ] }, "confidence": 0.95, "ambiguites": [] }

Code DRL généré automatiquement :

package rules.douane; rule "Calcul droits marchandise supérieure à 1M FCFA" salience 10 when $d : Declaration( valeur_marchandise > 1000000 ) then $d.setAction("CALCULER_DROITS"); update($d); end

7. Valeur ajoutée

Sans cette fonctionnalité, l'utilisateur devrait créer chaque règle métier manuellement, champ par champ, avec une connaissance technique du format DRL. Grâce au NLP, il suffit d'écrire la règle en langage naturel — comme on la formulerait à un collègue — et le système génère automatiquement la structure technique complète et le code exécutable. Cela rend le système accessible aux agents douaniers sans formation informatique.
