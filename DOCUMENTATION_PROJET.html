1. Vue d'ensemble du projet

Ce projet est une application backend Spring Boot dédiée à la gestion complète des processus douaniers d'importation et d'exportation. Il constitue le cœur métier d'une plateforme permettant aux agents douaniers, opérateurs et administrateurs de gérer l'ensemble du cycle de vie des dossiers douaniers.

📋

Règles métier

Création, versionning et évaluation des règles douanières (taxes, quotas, certifications)

⚙️

Workflow Camunda

Automatisation des processus d'import/export via moteur BPMN

🤖

Intelligence artificielle

Génération de règles métier depuis le langage naturel (Groq / LLaMA)

🔐

Keycloak

Authentification OAuth2/OIDC avec gestion des rôles

📁

Dossiers douaniers

Suivi des dossiers d'import et d'export avec statut en temps réel

📄

Rapports PDF

Génération automatique de rapports à partir des instances de processus

Frontend associé : Application Angular tournant sur le port 4200. Le backend expose ses API REST sur le port 8081 avec CORS configuré.

6. Couche Service

UserService

Gère les utilisateurs en synchronisant les données entre la base de données locale et Keycloak.

  • createUser() — Création dans Keycloak + BDD avec attribution de rôles multiples
  • updateUserKeycloak() — Synchronisation des modifications vers Keycloak
  • updateUserRoles() — Modification des rôles assignés
  • deleteUser() — Suppression dans Keycloak et en BDD
  • updateUserStatus() — Activation / désactivation du compte

KeycloakService

Hub d'intégration avec l'API Admin REST de Keycloak.

  • getAdminToken() — Acquisition du token OAuth2 administrateur
  • createUser() — Création dans le realm Keycloak
  • getAllUsersWithRoles() — Récupération des utilisateurs et rôles
  • toggleUserStatus() — Activation / désactivation
  • sendResetPasswordEmail() — Déclenchement du flux "mot de passe oublié"

RegleMetierService

Fonctionnalité clé : Chaque appel à update() génère automatiquement un snapshot immuable dans versions_regle avant d'appliquer la modification. Cela assure une piste d'audit complète et conforme.
  • create() — Création d'une règle avec ses conditions
  • update() — Mise à jour + snapshot de version automatique
  • toggle() — Activation / désactivation d'une règle
  • delete() — Suppression en cascade (conditions incluses)

NlpService

  • Construit un prompt spécialisé dans le domaine douanier
  • Appelle l'API Groq (modèle llama-3.3-70b-versatile)
  • Parse la réponse JSON du LLM → structure NlpRegleDto
  • Génère automatiquement la syntaxe Drools DRL
  • Retourne : règle structurée + DRL + score de confiance + ambiguïtés

DossierImportService / DossierExportService

  • Génération automatique du numéro de dossier (IMP-XXXXX / EXP-XXXXX)
  • Statut initial : EN_ATTENTE
  • Horodatage automatique des modifications

PdfService

  • Interroge l'historique Camunda pour une instance de processus
  • Génère un rapport PDF via OpenPDF
  • Retourne un flux binaire téléchargeable

10. Intégration Camunda BPM

Camunda gère les workflows complexes des processus d'import/export, permettant la définition de processus BPMN visuels, l'assignation de tâches humaines et le suivi de l'état de chaque dossier.

Configuration

camunda.bpm.admin-user.id=admin
camunda.bpm.admin-user.password=admin
camunda.bpm.history-time-to-live=180   # jours

Processus BPMN déployé

ID : process_export_corrected — Gestion des variables avec conversion de dates flexibles.

Cycle de vie d'une instance

1
Démarrage POST /api/processus/demarrer — Variables initialisées (importateur, valeur, produit, pays…)
2
Tâche humaine assignée GET /api/processus/tasks — Agent douanier voit les tâches en attente
3
Complétion de la tâche POST /api/processus/tasks/{id}/complete — Variables mises à jour, tâche suivante activée
4
Fin du processus Historique disponible → Rapport PDF téléchargeable via GET /api/process/pdf/{instanceId}

11. Intégration Drools (Moteur de règles)

Drools permet l'évaluation dynamique des règles métier en syntaxe DRL, sans recompiler l'application. Le bean DroolsConfig charge automatiquement les fichiers *.drl depuis classpath:rules/.

Exemple de règle DRL

rule "Taxe sur produits de luxe"
    when
        $d : DossierImport(valeur > 50000, typeProduit == "LUXE")
    then
        $d.setStatut(StatutDossier.EN_COURS);
        $d.setCommentaire("Taxe de luxe applicable - vérification requise");
end
Intégration NLP → Drools : Le service NLP génère automatiquement le code DRL à partir de phrases en langage naturel, prêt à être chargé dans le moteur Drools.

12. Intégration NLP — Intelligence artificielle

Architecture du flux NLP

1
Saisie utilisateur L'expert douanier saisit une phrase en français décrivant une règle métier
2
Prompt Engineering NlpService construit un prompt enrichi du contexte douanier tunisien/international
3
Appel API Groq Modèle llama-3.3-70b-versatile — génère une structure JSON de règle métier
4
Post-traitement Parsing JSON → NlpRegleDto + génération automatique du code DRL Drools
5
Réponse enrichie Règle structurée + code DRL + score de confiance + liste des ambiguïtés détectées

13. Génération de PDF

PdfService interroge l'API historique de Camunda pour récupérer toutes les variables d'une instance de processus terminée, puis génère un rapport PDF structuré via la bibliothèque OpenPDF.

Endpoint : GET /api/process/pdf/{processInstanceId}

Contenu du rapport

  • Numéro de dossier et type (import / export)
  • Informations déclarées (importateur/exportateur, produit, valeur, pays)
  • Historique des tâches et décisions prises
  • Statut final du dossier
  • Horodatage du rapport

16. Flux métier principaux

Scénario 1 — Création et versionning d'une règle

1
CréationPOST /api/regles → RegleMetierService.create() → Règle + conditions sauvegardées, Version 1 créée
2
ModificationPUT /api/regles/{id} → Snapshot de l'état actuel créé dans versions_regle → Nouvelle règle sauvegardée (Version 2)
3
AuditGET /api/regles/{id}/versions → Historique complet avec motifs de modification et snapshots des conditions

Scénario 2 — Workflow d'importation complet

1
Création du dossierPOST /api/import/dossiers → Numéro IMP-00001 généré → Statut : EN_ATTENTE
2
Démarrage CamundaPOST /api/import/demarrer → Variables passées, instance Camunda créée
3
Traitement par l'agentGET /api/import/taches → Agent voit ses tâches assignées
4
ValidationPOST /api/import/taches/{id}/completer → Variables mises à jour, tâche suivante déclenchée
5
Rapport PDFGET /api/process/pdf/{instanceId} → PDF téléchargeable avec historique complet

Scénario 3 — Génération de règle par IA

1
SaisieExpert douanier saisit une phrase décrivant une règle en français
2
Appel NLPPOST /api/nlp/convertir → NlpService appelle Groq API avec prompt spécialisé douanier
3
Réponse structuréeRègle JSON + DRL Drools + confidence 0.92 + ambiguïtés détectées
4
IntégrationExpert valide et sauvegarde via POST /api/regles → Règle intégrée au système
